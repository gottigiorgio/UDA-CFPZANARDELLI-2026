1. Cos'è la Paginazione?

Immagina di avere un database con 1000 ticket che vuoi mostrare sul tuo sito web. Se li mostrassi tutti insieme, la pagina sarebbe lunghissima, oltre che un problema da caricare

La paginazione è come dividere un libro in capitoli: invece di mostrare tutti i 1000 ticket, ne mostri solo 10, 20 o 30 per volta, e l'utente può navigare tra le pagine.

Esempio Visivo

Invece di vedere tutto questo:

Ticket 1, Ticket 2, Ticket 3, ... Ticket 998, Ticket 999, Ticket 1000

Vedi solo:

Ticket 1, Ticket 2, Ticket 3, ... Ticket 10

2. Come Funziona la Paginazione?

La paginazione si basa su tre numeri fondamentali:

  1. elementiTotali: quanti dati si ha in totale (es: 1000 ticket)
  2. elementiPerPagina: quanti elementi vuoi mostrare in ogni pagina (es: 10)
  3. paginaAttuale: quale pagina stai visualizzando (es: pagina 3)

Esempio Pratico

Totale Ticket Per Pagina Totale Pagine
1000 10 100 (1000 ÷ 10)
50 20 3 (50 ÷ 20 = 2.5 → arrotondato a 3 per eccesso)

3. Come lo implemento in Flask?

1Modificare la Route Python

Prima di tutto, devi modificare il file Python che gestisce la pagina (la "route"):

# Route Flask per la lista dei ticket @richieste.route('/tickets') def listTicket(): # Se 'page' non esiste nell'URL, usa 1 come default page = request.args.get('page', 1, type=int) # Se 'per_page' non esiste, usa 10 come default per_page = request.args.get('per_page', 10, type=int) # Pagina i dati dal database # Questa funzione fa tutto il lavoro pesante! pagination = Ticket.query.paginate( page=page, # quale pagina mostrare per_page=per_page # quanti elementi per pagina ) # Passa i dati al template HTML return render_template( 'list_ticket.html', tickets=pagination.items, # solo i ticket della pagina corrente pagination=pagination # tutte le info sulla paginazione )
Spiegazione riga per riga:

request.args.get('page', 1, type=int): Legge il parametro "page" dall'URL. Se non esiste, usa 1 come valore predefinito.

Ticket.query.paginate(): Questa è la funzione magica! Prende automaticamente solo i dati della pagina richiesta dal database.

pagination.items: Contiene solo i ticket della pagina corrente (es: i ticket da 11 a 20 se sei a pagina 2).

2Aggiungere il Selettore nel Template HTML

Questo permette all'utente di scegliere quanti elementi vedere per pagina:

<!-- Form per selezionare quanti elementi mostrare --> <form method="get"> <label for="per_page">Mostra:</label> <!-- Dropdown con le opzioni --> <select name="per_page" id="per_page" onchange="this.form.submit()"> <option value="10">10</option> <option value="20">20</option> <option value="50">50</option> </select> <!-- Campo nascosto per resettare alla pagina 1 --> <input type="hidden" name="page" value="1"> </form>
Come funziona:
Quando l'utente cambia la selezione, onchange="this.form.submit()" invia automaticamente il form. Questo aggiorna l'URL con il nuovo valore (es: da ?per_page=10 a ?per_page=20).

3Aggiungere i Pulsanti di Navigazione

Questi permettono di andare avanti e indietro tra le pagine:

<!-- Controlli di navigazione tra le pagine --> <nav aria-label="Navigazione pagine"> <ul class="pagination"> <!-- Pulsante PRECEDENTE --> <li class="page-item {% if not pagination.has_prev %}disabled{% endif %}"> <a class="page-link" href="{{ url_for('richieste.listTicket', page=pagination.prev_num, per_page=request.args.get('per_page', 10)) }}"> ◀ Precedente </a> </li> <!-- Informazione PAGINA CORRENTE --> <li class="page-item disabled"> <span class="page-link"> Pagina {{ pagination.page }} di {{ pagination.pages }} </span> </li> <!-- Pulsante SUCCESSIVO --> <li class="page-item {% if not pagination.has_next %}disabled{% endif %}"> <a class="page-link" href="{{ url_for('richieste.listTicket', page=pagination.next_num, per_page=request.args.get('per_page', 10)) }}"> Successivo ▶ </a> </li> </ul> </nav>

4. Il Flusso Completo (Passo per Passo)

1L'utente visita la pagina

URL: http://tuosito.com/tickets

Flask carica automaticamente la pagina 1 con 10 elementi (valori predefiniti).

2Flask interroga il database

La funzione paginate() fa una query SQL intelligente:

SELECT * FROM tickets LIMIT 10 OFFSET 0;

LIMIT 10: Prendi solo 10 record
OFFSET 0: Inizia dal primo record (record 0-9)

3L'utente clicca "Successivo"

URL diventa: http://tuosito.com/tickets?page=2

Flask fa una nuova query:

SELECT * FROM tickets LIMIT 10 OFFSET 10;

OFFSET 10: Salta i primi 10 record e mostra i successivi 10 (record 10-19)

4L'utente cambia da 10 a 20 elementi

URL diventa: http://tuosito.com/tickets?per_page=20&page=1

Flask fa una nuova query:

SELECT * FROM tickets LIMIT 20 OFFSET 0;

Ora mostra i primi 20 record invece di 10!

5. L'Oggetto Pagination

Quando chiami Ticket.query.paginate(), Flask ti restituisce un oggetto speciale con tante proprietà utili:

Proprietà Cosa Contiene Esempio
pagination.items I record della pagina corrente Lista di 10 ticket
pagination.page Numero pagina corrente 3
pagination.pages Numero totale di pagine 100
pagination.total Numero totale di record 1000
pagination.has_prev C'è una pagina precedente? True o False
pagination.has_next C'è una pagina successiva? True o False
pagination.prev_num Numero pagina precedente 2
pagination.next_num Numero pagina successiva 4

6. Perché Usare la Paginazione Lato Server?

Senza Paginazione

  • Carica TUTTI i 1000 ticket dal database
  • La pagina è lentissima
  • Consuma molta memoria
  • Esperienza utente pessima

Con Paginazione

  • Carica solo 10 ticket alla volta
  • La pagina è velocissima
  • Consuma poca memoria
  • Esperienza utente ottima

7. Riepilogo Finale

  1. La paginazione divide grandi quantità di dati in pagine più piccole
  2. Flask ha una funzione paginate() che fa tutto il lavoro pesante
  3. L'URL contiene i parametri page e per_page per sapere cosa mostrare
  4. Il database carica SOLO i dati necessari, non tutto
  5. L'oggetto pagination contiene tutte le informazioni per costruire la navigazione
Ora che conosci la teoria, prova a implementarla nel tuo progetto! Inizia con valori semplici (10 elementi per pagina) e sperimenta. La pratica è il miglior modo per imparare!
